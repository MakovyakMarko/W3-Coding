Posibnyk zi stylju HTML

Poslidovnyj, chystyj i akyratnyj HTML-kod polegshuje inshym chytannja ta rozyminnja vashogo kodu.
Osj kiljka vkazivok ta porad schodo stvorennja horoshogo HTML-kodu.

Завжди декларувати тип документа

Завжди оголошуйте тип документа як перший рядок у вашому документі. Правильний тип документа для HTML:
!DOCTYPE html

Використовуйте назви елементів у нижньому регістрі

HTML дозволяє змішувати великі та малі літри в назвах елементів. Проте ми рекомендуємо використовувати назви елементів у нижньому регістрі, оскільки

  • Змішування великих і малих імен виглядає погано
  • Розробники зазвичай використовують імена в нижньому регістрі
  • Нижній регістр виглядає чистіше
  • Писати в малому регістрі легше

Закрийте всі елементи HTML

У HTML вам не потрібно закривати всі елементи, однак ми рекомендуємо закрити всі елементи

Використовуйте назви атрибутів у нижньому регістрі

HTML дозволяє змішувати велику та маленькі літери в назвах атрибутів. Проте ми рекомендуємо використовувати назви атрибутів в нижньому регістрі, оскільки:

  • Змішування великих і малих імен виглядає погано
  • Розробники зазвичай використовують імена в нижньому регістрі
  • Нижній регістр виглядає чистіше
  • Писати в нижньому регістрі легше

Завжди беріть значення атрибутів в лапки

HTML дозволяє залишати значення атрибутів без лапок. Проте ми рекомендуємо вводити значення атрибутів у лапки, оскільки:

  • Розробники зазвичай беруть значення атрибутів у лапки
  • Значення в лапках легше читати
  • Ви повинні використовувати лапки, якщо значення містить пробіли

Завжди вказуйте альтернативний текст, ширину і висоту для зображень

Завжди вказуйте альтернативний текст для зображень. Цей атрибут важливий, якщо зображення з якихось причин не можливо відобразити.
Крім того, завжди вказуйте ширину та висоту зображень. Це зменшує мерехтіння, оскільки браузер може зарезервувати місце для зображення перед завантаженням.

Пробіли та знаки рівності

HTML допускає пробіли навколо знаків ріності. Але без простору легше читати і краще групувати об'єкти разом.

Уникайте довгизх рядків коду

При використанні редактора HTML не зручно прокручувати вправо та вліво, щоб прочитати html-код. Намагайтесь уникати занадто довгих рядків коду.

Порожні лінії та відступи

Не додавайте без причини порожні рядки, пробіли чи відступи. Для читабельності додайте порожні рядки до окремих великих або логічних блоків коду. Для читабельності додвійте два пробіли відступів. Не використовуйте клавішу табуляції.

Ніколи не пропускайте елемент title

Елемент title є обов'язковим у html.
Вміст заголовка сторінки дуже важливий для пошукової оптимізації (SEO)! Заголовок сторінки використовується алгоритмами пошукових систем для визначення порядку під час розміщення сторінок у результатах пошуку.
Елемент title:

  • Визначає назву на панелі інструментів браузера
  • Надає заголовок для сторінки, коли вона додається до вибраного
  • Відображає заголовок сторінки в результатах пошуку

Отже постарайтесь зробити назву якомога точнішою та змістовною

Опускаєте html і body?

HTML-сторінка буде перевірена без тегів html i body. Проте ми настійно рекомендуємо завжди додавати теги html i body! Пропуск body може призвести до помилок у старих браузерах. Пропуск html і body також може призвести до збою програмного забезпечення DOM i XML.

Опускаєте head?

Тег HTML head також можна опустити. Браузери додадуть усі елементи перед body, до елемента за замовчуванням head. Проте ми рекомендуємо використовувати head тег.

Закрити порожні елементи HTML?

У HTML необов'язково закривати порожні елементи. Якщо ви очікуєте, що програмне забезпечення XML/XHTML отримає достув до вашої сторінки залиште закриваючу похилу риску, оскільки вона є обов'язковою в XML та HTML.

Додайте атрибут lang

Ви завжди повинні включати lang атрибут всередині html тегу, щоб оголошувати мову веб-сторінки. Це покликане допомогти пошуковим систумам і браузерам.

Метадані

Щоб забезпечити належну інтерпритацію та правильне індексування пошуковою системою, мову кодування символів слід визначати якомога раніше в HTML-документі: meta charset="charset"

Налаштування вікна перегляду

Область перегляду - це видима область веб-сторінки для користувача. Це залежить від присрою - на мобільному телефоні вона буде меншою, ніж на екрані комп'ютера.
Ви повинні включити такий meta елемент на всі ваші веб-сторінки:
meta name="viewport" content="width=device-width, initial-scale-1.0"
Це дає браузаеру інструкції щодо того, як керувати та маштабувати сторінки.
Частина width=device-width встановлює ширину сторінки відповідно до ширини екрана пристрою (яка буде змінюватись залежно від присторю).
Частина initial-scale=1.0 встановлює початковий рівень маштабування, коли сторінка вперше завантажується браузером.

Коментарі HTML

Короткі коментарі слід писати в рядок. Коментарі, які охоплюють більше рядка слід охопити відкриваючими і закриваючими тегами коментаря. Довгі коментарі легше спостерігати, якщо вони мають відступ двома пробілами.

Використання таблиць стилів

Використовуйте простий синтаксис для посилань на таблиці стилів (type атрибут не потрібен): link rel="stylesheet" href="styles.css" Короткі правила CSS можна написати стиснутими, наприклад:
p.intro {font-family:Verdana;font-size:16em;} Довгі правила CSS слід записувати в кілька рядків: 

                
                    body{
                        background-color: lightgrey;
                        font-family: "Arial Black", Helvetica, sans-serif;
                        font-size: 16em;
                        color: black;
                    }

  • Помістіть відкриваючу дужку на ту ж лінію, що й селектор
  • Використовуйте один пробіл перед дужкою, що відкривається
  • Використовуйте два пробіли для відступів
  • Використовуйте крапку з комою після кожної пари властивість-значеня, включаючи останню
  • Використовуйте лапки навколо значень, лише якщо значення містить пробіли
  • Помістіть закриваючу дужку на новий рядок, без пробілів

Завантаження JavaScript в HTML

Використовуйте простий синтаксис для завантаження зовнішніх скриптів (type атрибут не потрібен):
script src="myscript.js"

Доступ до елементів HTML за допомогою JavaScript

Використання "неохайного" html-коду може призвести до помилок JavaScript.
Ці два оператори JavaScript дадуть різні результати: 

                
                    getElementById("Demo").innerHTML = "Hello";

                    getElementById("demo").innerHTML = "Hello";

Використовуйте імена файлів у нижньому регістрі

Деякі веб-сервери (Apach, Unix) чутливі до імен файлів: "london.jpg" неможливо отримати як "London.jpg"
Інші веб-сервери (Microsoft, IIS) не чутливі до регустру: "london.jpg" можна отримати як "London.jpg"
Якщо ви використовуєте поєднання великих і малих регістрів, ви повинні знати про це.
Якщо ви перейдете від сервера, що не чутливий до регістру, до сервера, який чутливий до регістру, навіть невеликі помилки пошкодять вашу мережу!
Щоб уникнути цих проблем, завжди використовуйте імена файлів у нижньому регістрі!

Розширення файлів

Файли HTML повинні мати розширення .html ( допускається розширення .htm).
Файли CSS повинні мати розширення .css .
Файли JavaScript повинні мати розширення .js .

Відмінності між .htm i .html

Немає різниці між цими розширеннями. Обидва будуть розглядатися як HTML будь-яким веб-браузером і веб-сервером.

Імена файлів за замовчуванням

Якщо URL-адреса не вказує назву файлу в кінці (наприклад, "https://www.w3schools.com/"), сервер просто додає ім'я файлу за замовчуванням, наприклад "index.html","index.htm", "default.html", "deafault.htm".
Якщо ваш сервер налаштовано лише на "index.html", як ім'я файлу за замовчуванням, ваш файл має називатися "index.html", а не "default.html".
Однак сервери можна налаштувати з більш ніж одним іменем файлу за замовчуванням; зазвичай ви можете встановити скільки завгодно імен файлів за замовчуванням.
