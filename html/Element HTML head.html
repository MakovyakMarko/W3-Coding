title, style, meta, link, script, base

Element HTML head

Element head je konteinerom dlja metadannyh (danyh pro dani) i rozmischuetjsja miz html tegom i body tegom.
Metadani HTML - ce dani pro dokumenty HTML. Metadani ne vidobrazajutjsja.
Metadani zazvychaj vyznachajutj nazvu dokymenta, nabir symvoliv, styli, scenariji ta inshu metainformaciju.

Element HTML title

Element title vyznachaje nazvu dokumenta. Zagolovok maje buty lyshe takstovym i vidobrazaetjsja v rjadku zagolovka brauzera abo na vkladci storinky.
Element title je obovjazkovym u dokumentah HTML!
Vmist zagolovka storinky duze vazlyvyj dlja poshukovoji optymizaciji (SEO)! Zagolovok storinky vykorystovujetjsja algorytmamy poshukovyh system dlja vyznachennja porjadku pid chas rozmischennja storinok u rezuljtatah poshuku.
Element title:

  • vyznachae nazvu na paneli instrymentiv brauzera
  • Nadaje zagolovok dlja storinky,koly vona dodajetjsja do vybranogo
  • vidobrazaje zagolovok storinky v rezuljtatah poshukovoji systemy

Tomu postarajtesja zrobyty nazvu jak umoga tochnishoju ta zmistovnoju!

Element style

Element style vykorystovujetjsja dlja vyznachennja informaciji pro stylj dlja odnijeji storinky html:
style
body {backgound-color:powderblue;}
h1 {color:red;}
p {color:blue;}

Element link

Element link vyznachaje zvjazok miz potochnym documentom i zovnishnim resursom.
Teg link najchastishe vykorystovujetjsja dlja posylanj na zovnishni tablyci styliv:
link rel="stylesheet" href="mystyles.css"

Element HTML meta

Element meta zazvychaj vykorystovuetjsja dlja vyznachennja naboru symvoliv, opysu storinky, kljuchovyh sliv, avtora dokumenta ta nalashtuvanj oblasti peregljadu.
Metadani ne vidpbrazatymutjsja na storinci, ale vykorystovujutjsja brauzeramy (jak vidobrazyty vmist abo perezavantazyty storinku), poshukovymy systemamy (kljuchovi slova) ta inshymy web-servisamy.
Pruklady Vyznachte vykorystovuvanyj nabir symvoliv: meta charset="UTF-8"
Vyznachte kljuchovi slova dlja poshukovyh system: meta name="keywords" content="HTML, CSS, JavaScript"
Vyznachte opys vashoji web-storinky: meta name="description" content="Free Web tutorials"
Vyznadchte avtora storinky:
meta name="author" content="Marko Makovyak"
Onovljuvaty dokument kozni 30 sekund: meta http-equiv="refresh" contant="30"
Nalashtuvannja oblasti peregljadum shob vash web-sajt vygljadav dobre na vsih prystrojah: meta name="viewport" content="width=device-width, initial-scale=1.0"

Nalashtuvannja vikna peregljadu

Oblastj peregljadu - ce vydyma oblastj web-stroinky dlja korystuvacha. Ce zalezutj vid prystroju - na mobiljnomu telefoni van bude menshym, niz na ekrani kompjutera.
Vy povunni vkljuchyty takyj meta element na vsi vashi veb-storinky: meta name="viewport" content="width=device-width, initial-scale=1.0"
Ce daje brauzeru instrukciji schodo togo, jak keruvaty rozmiramy ta mashtabuvannjam storinky.
Chastyna width=device-width vstanovljuje shyrynu storinky vidpovidno do shyryny ekrana prystroju (jaka bude zminjuvatysj zalezno vid prystroju).
Chastyna initial-scale=1.0 vstanovljuje pochatkovyj rivanj mashtabuvannja, koly storinka vpershe zavantazhujetjsja brauzerom.

Element HTML script

Element script vykorystovujetjsja dlja vyznachennja JavaScript na storoni klijenta.
Nastupnyj JavaScript pyshe "Pryvit, JavaScript!" v elements HTML z id="demo":
script
function myFunction() {
document.getElementById("demo").innerHTML="Hello, JavaScript!";
}

Elemetn HTML base

Elenent base vyznachaje bazovy url-adresu ta/abo ciljovu dlja vsih vidnosnyh url-adres na storinci.
Teg base povynen maty atrybut href abo ciljovyj atrybut, abo obydva. U dokumenti moze buty lyshe odyn base element!

base href="https://www.w3schools.com/" target="_blank"
