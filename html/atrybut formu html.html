Атрибут форми HTML

У цьому розділі описані різні атрибути для <form> елемента HTML

Атрибут action

Атрибут action визначає дію, яку потрібно виконати під час насилання форми.
Зазвичай дані форми надсилаються у файл на сервері, коли користувач натискає кнопку відправки.
У наведеному нижче прикладі дані форми надсилаються у файл під назвою "action_page.php". Цей файл містить сценарії на стороні сервера, який обробляє дані форми:

<form action="/action_page.php">
<label for="fname">Ім'я:</label>
<input type="text" id="fname" name="fname" value="John">
<label for="lname">Фамілія:</label>
<input type="text" id="lname" name="lname" value="Doe">
<input type="submit" value="Відправити">
</form>

Атрибут target

Атрибут target визначає, де відображати відповідь, отриману після відправлення форми.
Атрибут target може мати одне з таких значень:

Значення Описання
_blank Відкрити в новому вікні чи вкладці
_self Відкрити в поточному вікні
_parent Відкрити в батьківському фреймі
_top Відкрити в повному обсязі вікна
framename Відкрити в фреймі, за ім'ям фрейму

Значення за замовчуванням _self означає, що відповідь відкриється в поточному вікні.

<form action="action_page.php" target="_blank">

Атрибут method

Атирбут method визначає метод HTTP, який буде використовуватись під час подання данних форми.
Дані форми можна надіслати як змінні URL-адреси (за допомогою method="get") або як трензакцію повідомлення HTTP (за допомогою method="post").
Методом HTTP за замовчуванням під час подання даних форми є GET.

<form action="action_page.php" method="get">
<form action="action_page.php" method="post">

Примітки щодо GET:

  • Додає дані форми до URL-адреси в парах ім'я/значення
  • НІКОЛИ не використовуйте GET для надсилання конфіденсійних данних! (Дані поданої форми видно в URL-адресі!)
  • Довжина URL-адреси обмежена (2048 символів)
  • Корисно для подання форм, коли користувач хоча додати результат у закладки
  • GET добре підоходить для незахищених даних, як-от рядків запиту в Google

Примітки щодо POST:

  • Додає дані форми в тіло запиту HTTP (надані дані форми не відображаються в URL-адресі)
  • POST не має обмежень за розміром і може використовуватися для надсилання великих обсягів даних.
  • Надсилання форми за допомогою POST не можна додати до закладок.

Порада: завжди використовуйте POST, якщо дані форми містять конфіденційну інформацію або особисту інформацію!

Атирбут autocomplite

Атрибут autocomplite визначає, чи має бути увімкнено чи вимкнено атозаповнення форми.
Коли атозаповнення ввімкнено, браузер автоматично заповнює значення на основі значень, які користувач ввів раніше.

<form action="/action_page.php" autocomplite="on">

Атрибут novalidate

Атрибут novalidate є логічним атрибутом.
Якщо він присутній, він вказує, що дані форми (вхідні дані) не повинні перевірятися під час посилання.

<form action="/action_page.php" novalidate>
