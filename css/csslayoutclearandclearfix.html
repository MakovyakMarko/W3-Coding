Макет CSS - clear i clearfix

Властивість clear

Коли ми використовуємо float властивість і нас потрібно настуний елемент нижче (не праворуч чи ліворуч), нам доведеться використовувати clear властивість.

Властивість clear визначає, що має статись з елеменом, який знаходиться поруч із плаваючим елементом.

Властивість clear може мати одне з таких значень:

  • none - елемент не проштовхується нижчу лівих або правих плаваючих елементів. Це за замовчуванням.
  • left - елемент виштовхується нижче лівих плаваючих елементів
  • right - елемент виштовхується нижчу правих плаваючих елементів
  • both - елемент виштовхується нижче лівого та правого плаваючих елементів
  • inherit - елемент успадковує значення clear від свого батька

Коли очищеєте плаваючі елементи, ви повинні зіставити clear з плаваючим: якщо елемент плаває ліворуч, ви повинні очистити ліворуч. Ваш плаваючий елемент продовжуватиме плавати, але очищений елемент відображатиметься під ним на веб-сторінці.

Приклад. Цей приклад очищає поплавок ліворуч. Тут це означає, що елемент div2 розміщено нижче лівого плаваючого елемента div1:

.div1{
float:left;
}
.div2{
clear:left;
}

Злом clearfix

Якщо плаваючий елемент вищий за елемент, що містить, він "переповниться" за межі свого контейнера. Потім ми можемо додати хак clearfix, щоб вирішити цю проблему:

.clearfix{
overflow:auto;
}

Clearfix overflow:auto працює добре, доки ви можете контролювати поля та відступи (інакше ви можете побачити смуги прокручування). Однак новий сучасний хак clearfix безпечніший у використанні, і для більшості веб-сторінок використовується наступний код:

.clearfix::after{
content: "";
clear:both;
display:table;
}
