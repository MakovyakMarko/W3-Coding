HTML Uniform Resurs Locator (URL)

URL-адреса - це "наступне слово" для веб-адреси.
URL-адреса може складатися язі слів (наприклад, w3schools.com) або адреси інтернет-протоколу (IP) (наприклад, 192.68.20.50)
Більшість людей вводять ім'я під час серфіегу, тому що імена легше запам'ятати, нід цифри.

URL - уніфікований локатор ресурсів

Веб-ресурси запитують сторінки з веб-серверів за допомогою url-адреси.
Уніфікований локатор ресурсів (URL) використовується для адресації документа (вбо інших данних) в Інтернеті.
Веб-адреса, як-от https://www.google.com.ua/html/file1.asp, відповідає таким правилам синтаксису:
scheme://prefix.domain:port/path/filename

  • sheme - визначає тип інтернет-сервісу (найпоширенішим є http або htpps)
  • prefix - визначає префікс домену (за замовчуванням для http є www)
  • domain - визначає доменне ім'я інтернету ( наприклад google.com.ua)
  • port - визначає номер порту на хості (за замовчуванням для http є 80)
  • path - визначає шлях на сервері (якщо опущено: корневий каталог сайту)
  • filename - визначає назву документу або ресурсу

Поширені схеми URL-адрес

У таблиці нище, наведено деякі поширені схеми:

Scheme Short for Used for
http Hyper Text Transfer Protocol Common web pages. Not encryoted
https Secure Hyper Text Transfer Protocol Secure web pages. Encrypted
ftp File Trasfer Protocol Downloading or Uploading files
file   A file on your computer

Кодування URL

URL-адреси можна надсилати лише через Інтернет, використовуючи набір символів ASCII. Якщо url-адреса містить символи за межами набору ASCII, url-адресу потрібно перетворити.
Кодування url-адреси перетворює сиволи відмінні від ASCII, у формат, який можна передавати через Інтернет.
Кодування URL-адреси замінює символи, відмінні від ANCII, на %, зя якими слідують шіснадцяткові цифрию URL-адреси не можуть містити пробіли. Кодування url-адреси зазвичай замінює пробіли знаком плюс (+) або %20
