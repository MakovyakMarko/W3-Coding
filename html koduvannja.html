Від ASCII до UTF-8

ASCII був першим стандартом кодування символів. ASCII визначив 128 різних символів, які можна використовувати в Інтернеті: цифри (0-9), англійські літри (A-Z) і деякі спеціальні символи, як-то ! $ + - ( ) @ < >
ISO-8859-1 був стандартним набором символів для HTML 4. Цей набір символів підтримував 256 різних кодів символів. HTML 4 також підтримує UTF-8.
ANSI (Windows-1252) був оригінальним набором символів Windows. ANSI ідентичний ISO-8859-1, за винятком того, що ANSI має 32 додаткові символи.
Специфікація HTML 5 заохочує веб-розробниуів використовувати набір символів UTF-8 який охоплює майже всі символи у світі!

Атрибут кодування HTML

Щоб правильно відображати сторінку HTML, веб-браузер повинен знати набір символів, який використовується на сторінці. Це вказано в meta тегу: meta charset="UTF-8"

Відмінності між наборами символів

Набір символів ASCII

ASCII використовує значення від 0 до 31 ( і 127) для контрольних символів.
ASCII використовує значення від 32 до 126 для літер, цифр і символів.
ASCII не використовує значення від 128 до 255.

Набір символів ANSI (Windows-1252)

ANSI ідентичний ASCII для значень від 0 до 127.
ANCI має власний набір символів для значень від 128 до 159.
ANCI ідентичний UTF-8 для значень від 160 до 255.

Набір символів ISO-8859-1

ISO-8859-1 ідентичний ASCII для значень від 0 до 127.
ISO-8859-1 не використовує значення від 128 до 159.
ISO-8859-1 ідентичний UTF-8 для значень від 160 до 255.

Набір символів UNT-8

UTF-8 ідентичний ASCII для значень від 0 до 127.
UTF-8 не використовує значення від 128 до 159.
UTF-8 ідкетичний як ANSI, так і 8859-1 для значень від 160 до 255.
UTF-8 продовжується зі значення 256 з більш нід 10 000 різних символів.
